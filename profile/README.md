---

## What We Build

- Backend systems  
- APIs & microservices  
- Data pipelines

- Infrastructure-ready services  

---

## Engineering Principles

- Code quality over shortcuts  
- Simple systems scale better  
- Readability is non-negotiable  
- Tests are standard, not optional  
- Architecture before features  

---

## Repository Standards

Each project includes:

- Clear purpose  
- Architecture overview  
- Setup instructions  
- Usage examples  
- Key design decisions  

Naming:

- `kebab-case` or `snake_case`  
- Specific and descriptive  

---

## Code Standards

- PEP 8 compliant  
- Type hints required  
- Docstrings for public APIs  
- Tests included by default
